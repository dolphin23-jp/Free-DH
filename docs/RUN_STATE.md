# RUN_STATE — 周回ステートマシン

T12で作成した周回状態をT18で15戦の完全な周回へ統合した。戦闘ルール自体は `COMBAT_SPEC.md`、数値は `src/data/config.json` を正とする。

## 状態遷移

```text
idle
  └─ startRun(seed, inventory?, abyssLevel?) → preBattle
       └─ beginBattle() → battle
            ├─ 通常敵に勝利 → ドロップ2枠 → preBattle（battleIndex + 1）
            ├─ ボスに勝利 → ドロップ3枠 → bossReward
            │    ├─ 拡張 + 回復 → 次エリアのpreBattle / 最終ボスならresult
            │    └─ 拡張 + 追加ドロップ2枠 → 宝箱処理 → 次エリア / result
            └─ 敗北 → result: defeated

result
  ├─ startRun(seed) → 新しい周回
  └─ resetRun() → idle
```

`battleIndex` は内部では0始まりで0〜14を取る。画面上の「第N戦」は `battleIndex + 1` とする。`battlesWon` は勝利済み戦闘数であり、敗北時には増えない。ボス撃破後は報酬処理が完了するまで同じ `battleIndex` を維持する。

## 敵順と戦闘シード

- 各エリアの通常敵4体を `fork(runSeed, "enemyOrder:{area}")` の独立ストリームでFisher–Yatesシャッフルする。
- 各エリアの5戦目にはそのエリアのボスを固定する。
- 3エリアを連結し、合計15戦の `enemyOrder` とする。
- 各戦闘の乱数シードは `fork(runSeed, "battle:{battleIndex}")` から導出する。

同じ `runSeed` では敵順と各戦闘シードが完全に一致する。`Math.random` は使用しない。

## ボス報酬

### カバン拡張

各ボスは1回の拡張キットを与える。現在のカバンが設定上限6×4に達していない方向から、`+1列` または `+1行` を選択する。どの順番で選んでも3ボス後に6×4へ到達できる。

受け入れテストの基準経路は以下。

```text
4×3 → 5×3 → 5×4 → 6×4
```

ショップ専用の最終拡張6×4→6×5はボス報酬では選択できない。

### 回復または追加ドロップ

拡張と同時に以下のいずれかを選ぶ。

- 最大HPの40%を回復する（最大HPを超えない）。
- 同じ `drops:{battleIndex}` ストリームから続く2枠を開封する。

ボス戦では最初から5枠分を1本のストリームで生成し、先頭3枠を通常ボスドロップ、末尾2枠を選択報酬として予約する。回復を選んだ場合は予約分を破棄する。これにより選択前後で乱数系列を作り直さず、決定論を維持する。

天井カウンタは1戦につき最大1回だけ増える。追加2枠に橙が含まれる場合は、追加ドロップを選択した時点で天井を0へ更新する。

## リザルトと魂片

敗北・クリア時の `RunResult` は以下を保持する。

- 勝利数
- 到達戦闘indexと到達戦闘数
- 最終HP / 最大HP / 所持G
- その周回で獲得する魂片数

魂片は設定値から次式で算出する。

```text
ceil(到達戦闘数 × (1 + 0.2 × 深淵Lv)) + クリア時10
```

深淵0クリアは `15 + 10 = 25`。T18ではリザルトへ算出値を載せ、所持魂片への加算と恒久保存はT20で実装する。

## 周回中に持ち越す状態

- 現在HP / 最大HP
- 所持G
- 選択中の深淵Lv
- カバンの列数・行数と配置アイテム
- ストレージ容量と格納アイテム
- アイテムのアフィックス解決値
- `runScalingDamage` のアイテム別加算値
- 勝利数と現在戦闘index
- 未処理のボス報酬と、追加ドロップ選択済み状態

ブロック、スタミナ、状態異常、戦闘中だけのバフは戦闘状態に属し、周回ストアへは保存しない。

## 戦闘エンジンとの境界

`selectCurrentCombatSetup` は現在の周回状態から `CombatSetup` を生成する。カバン内アイテムだけをbuildへ変換し、現在HP・最大HP・所持Gと当該戦闘シードを渡す。

戦闘終了後は `battleResolutionFromCombatState` で以下を周回側へ戻し、`completeBattle` へ渡す。

- 勝敗
- 戦闘後HP / 最大HP
- 戦闘後所持G
- 物理アイテムごとの `runDamageBonus`

仮想複製は周回インベントリへ保存しない。

## スナップショットとリロード

`exportRunSnapshot` はactionを含まないJSON直列化可能なversion 2スナップショットを返す。`createRunStore(snapshot)` または `loadSnapshot(snapshot)` で復元する。

復元時にはversion、HP、深淵Lv、戦闘index、敵順とrunSeedの一致、インベントリ、リザルト、ボス報酬状態を検証する。未選択のボス報酬と、追加ドロップを選択した後の状態のどちらも復元可能。

ブラウザの完全な自動保存、JSONエクスポート／インポート、セーブ移行はT20で実装する。

## インベントリUIとの境界

周回ストアはインベントリの所有状態と直列化を担当する。アイテム形状、回転後の占有マス、カバン境界、重なり拒否、ドラッグ操作はカバンUIの責務であり、周回ストアでは二重実装しない。
