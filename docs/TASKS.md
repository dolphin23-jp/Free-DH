# TASKS.md — 実装タスク分割(受け入れ条件付き)

> 原則: **上から順に、1タスク=1セッション/1PR。** 受け入れ条件(AC)を全て満たすこと。
> 共通の完了条件は AGENTS.md の Definition of Done。

## Phase 0: 基盤とエンジン

### T00 リポジトリ雛形
Vite+React+TS(strict)+Vitest+ESLint/Prettier+pnpm。GitHub Actions(PR時に test/lint/build)。Vercel用設定。
**AC**: `pnpm dev/test/lint/build` が全て成功。CIがPRで走る。README に起動手順。

### T01 データ層と型
`src/data/` に items/enemies/recipes.json を配置(リポジトリに同梱済みのファイルを使用。数値を変えないこと)。COMBAT_SPEC §10 の設定値から `config.json` を起こす。zod でスキーマ検証し型を導出。
**AC**: `tests/data.test.ts` — 全JSONがスキーマ適合 / アイテム78(通常60+融合18)/ ID一意 / レシピ20の参照先が全て実在 / 融合限定18種すべてに到達レシピがある。

### T02 乱数モジュール
mulberry32 + `fork(seed, label)`(COMBAT_SPEC §2)。
**AC**: 既知シードに対する出力列の固定ベクトルテスト。fork の独立性テスト。

### T03 エンジン骨格(tickループ)
状態モデル、tickパイプライン(SPEC §3)、CD管理、スタミナ回復、発動順序(§4: グリッド順・ready待機)。効果は damage/block のみ仮実装。
**AC**: 「CD0でスタミナ不足のアイテムが、回復後にグリッド順で発動する」ことを検証するテスト。1000tickで状態が破綻しない。

### T04 ダメージパイプライン
SPEC §5 の (a)〜(g)。加算→乗算合算→会心→特効→端数→ブロック/pierce→命中後処理。CD減合算上限-60%・CD下限0.3s。
**AC**: 乗算修正の合算仕様(+30%と+50%=×1.8)のテスト。pierce・会心倍率上書き(戦斧2.5)・damageReduction のテスト。

### T05 状態異常
毒(永続・整数秒)/ 火傷(3秒バッチ)/ 鈍化。Trueダメージ規則(ブロック無視・トリガー/反射/吸血を発生させない)。cleanseSelf。
**AC**: 整数秒タイミングのテスト。火傷バッチの独立消滅テスト。Trueダメージが反射されないテスト。

### T06 トリガーシステム
battleStart / onHit / onKill / onDamaged / onBlocked / battleWin / hpBelow。連鎖深度1。封印の最優先解決。
**AC**: 「反射が反射されない」「onHitからのダメージがonHitを再誘発しない」テスト。hpBelowが1回だけ発火するテスト。

### T07 隣接システム
グリッド幾何(占有マス・上下左右 / C14のみ8方向)、対象タグ照合、戦闘開始時スナップショット(封印・複製の後)。
**AC**: 2×3アイテムの全隣接マス判定テスト。封印されたアイテムの隣接バフが無効になるテスト。

### T08 スペシャルキーワード(アイテム側)
SPEC §8 のうち: openingShot / execute / battleScalingDamage / runScalingDamage / poisonFinisher / guardianHeal / sealAdjacent / duplicateAdjacent / readyAllCooldowns / runMaxHpOnKill / dropLuck / healPercentOnWin / undeadSlayer。
**AC**: キーワードごとに最低1テスト。duplicateAdjacent の決定的選択(最高レア→グリッド順)テスト。

### T09 敵トレイト
hpRegen / blockRegen / revive / lifesteal(disabledBy:burn)/ goldSteal+fleeAfterHits / openingFrenzy / enrage / staminaDrain / thorns / sealRandomItem / phaseTransition(魔王: バフ・毒持ち越し)。サドンデス。
**AC**: 敵15種それぞれに参照ビルドとの対戦テスト(ENEMIES.md の検算例と概ね一致)。野盗の逃走が「勝利・G消失」になるテスト。魔王フェーズ移行で毒が持ち越されるテスト。

### T10 simulate API + イベントログ + ゴールデンテスト
`simulate(build, enemyId, seed) → { result, events, stats }`(SPEC §11)。
**AC**: 固定 build×enemy×seed 3組のイベントログハッシュをスナップショット固定。同一入力2回で完全一致。

### T11 ヘッドレスシミュレータCLI
`pnpm sim` — ビルド定義×敵×N回で勝率/平均戦闘時間/平均被ダメを出力。`src/sim/builds/` に「標準ビルド」(エリア1/2/3相当)を定義。
**AC**: ENEMIES.md §9 の5項目が計測できる。目標帯からの逸脱を警告表示(テスト失敗にはしない)。

## Phase 1: ゲームとして遊べる形

### T12 周回ステートマシン
Zustand: runSeed / 戦闘index / HP持ち越し / 所持G / カバン&ストレージ / 敵順(enemyOrderストリーム)。開始→15戦→クリア/敗北→リザルトの遷移。
**AC**: 状態遷移の単体テスト。リロードで周回が壊れない(暫定: メモリのみで可)。

### T13 カバンUI(dnd-kit)
グリッド配置・回転・衝突判定・ストレージ(8枠)・売却。配置=エンジン入力のbuildに変換。
**AC**: 2×3アイテムの配置/回転/重なり拒否が動く。タッチ操作(スマホ)で配置可能。

### T14 戦闘ビュー(ログ再生)
イベントログを1x/2x再生。HPバー・ブロック表示・ダメージ数字・発動ハイライト。**UI側で計算しない。**
**AC**: 同一シードで再生結果が simulate の result と一致。倍速切替が動く。

### T15 ドロップ体験
勝利→宝箱→タップ開封(1個ずつ)→レア度色演出(仮でよい)。レア度抽選(エリア/深淵テーブル+dropLuck)、天井カウンタ(周回跨ぎ保存)、アフィックス付与。
**AC**: 抽選が dropsストリームで決定論的。天井: 橙30戦未出で+1pp/戦、入手でリセットのテスト。

### T16 ショップ
6枠陳列・リロール(5→7→10→14→19G)・購入・売却(秤/sellOverride反映)・回復サービス(20G/HP20/1回)。
**AC**: リロール価格系列とショップごとのリセットのテスト。shopストリームで決定論的。

### T17 敵予告と図鑑
次戦予告(初遭遇は「???+ヒント」、遭遇済はフル表示)。図鑑画面(アイテム発見状況・敵・レシピの?シルエット)。
**AC**: 初遭遇→登録→表示切替のテスト。図鑑データがセーブに含まれる。

### T18 周回ループ統合
ボス報酬(拡張キット: +1列/+1行選択、二択: HP40%回復 or 追加ドロップ2枠)。敗北/クリアのリザルト画面(到達・獲得魂片)。
**AC**: 通しで1周プレイ可能。拡張系列 4×3→5×3→5×4→6×4 のテスト。

### T19 融合
隣接ペアがレシピ一致で光る→確認ダイアログ→融合。発見済みレシピの図鑑登録。
**AC**: R01〜R20全ての成立テスト(順序不問)。誤爆しない(確認必須)。

### T20 メタ進行とセーブ
魂片獲得式・アンロックショップ(unlockCost)・深淵Lv選択(クリアで+1解放、敵係数適用、Lv3+の強化個体置換)。localStorage保存+JSONエクスポート/インポート。セーブスキーマにversionを付ける。
**AC**: 獲得式のテスト(深淵0クリア=25 等)。ロック中アイテムが抽選に出ないテスト。セーブ→リロード→完全復元。

### T23 バランスパス
`pnpm sim` の結果を基に config/items/enemies の数値のみを調整し、ENEMIES.md §9 の目標帯に収める。深淵Lv5・10相当でも実行し、直接攻撃型とDoT/回復/反射型の敵で被ダメ・勝率の乖離が広がりすぎていないか確認する(COMBAT_SPEC §13.21)。乖離が大きい場合は既存の攻撃倍率の対象effectを広げず、`config.json` に独立した「深淵trait倍率」ノブを新設する(この場合のみコード変更を伴ってよい)。
**AC**: 全敵の勝率・時間・被ダメが目標帯内(シミュレータレポートをPRに添付)。深淵Lv5・10でのタイプ別乖離を確認済み。通常の数値調整はコード差分ゼロ(JSONのみ)。
