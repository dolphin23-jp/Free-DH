# SPEC_TODO T20 — 深淵と強化個体の未確定事項

## TASKS T20: 強化個体の選出と倍率適用範囲

- **箇所**: T20は「Lv3+の強化個体置換」「敵係数適用」を要求するが、強化個体の数、対象戦、選出ストリーム、攻撃倍率をどの敵効果へ適用するかを指定していない。
- **問題**: 全通常敵を強化するか各エリア1体だけにするか、ボスを含めるか、毒・棘・金貨盗難・回復・累積強化まで攻撃倍率を掛けるかで難易度と固定シード結果が大きく変わる。
- **暫定判断**: 深淵Lv3以上では各エリアの通常4戦から1戦だけを `fork(runSeed, "elite:{area}")` で選び、ボスは除外する。深淵HP倍率を全敵へ、深淵攻撃倍率をability内の直接 `damage` 効果へ適用する。強化個体はそのHPへさらに設定値の+30%を乗算し、ドロップを1枠増やす。状態異常付与量、棘、再生、enrage加算値、goldSteal、revive値などは倍率対象にしない。
- **理由**: 既存JSONの効果語彙を変えず、何が「攻撃」かを明確に限定しつつ、同じシードで強化個体の位置を再現できる。各エリア1体ならLv3到達時の変化を見せながら、全戦が一律に強化される急激な難度上昇を避けられる。
- **今後の確認**: 強化個体を別enemy IDへ置換するか、出現数を深淵Lvで増やすか、trait・状態異常・フェーズ移行後の能力にも攻撃倍率を適用するかを正式決定する。変更時は深淵固定テスト、敵図鑑表示、ドロップ枠、セーブ再現性を同時に更新する。

## TASKS T20: 魂片報酬の重複防止キー

- **箇所**: リザルト画面で魂片を恒久所持へ反映するタイミングと、同一リザルトの再読込時の重複防止方法が指定されていない。
- **問題**: 画面描画時に無条件加算すると、Reactの再描画やリロード、JSONの再インポートで同じ報酬を繰り返し受け取れる。
- **暫定判断**: `runSeed + outcome + battlesWon + reachedBattleCount` を受取キーとして恒久保存し、未受取キーの場合だけ魂片と深淵解放を反映する。
- **理由**: 現在のRunResultだけで決定的に作成でき、セーブ／リロード後も報酬処理を安全に再実行できる。
- **今後の確認**: 同じrunSeedで意図的に複数回プレイする正式UIを追加する場合、周回開始時に永続的なrun IDを発行し、報酬キーをそのIDへ移行する。
