## 1. コンセプト

**「拾った戦利品をカバンに詰め込む」だけで戦う、王道ファンタジーのローグライト・オートバトラー。**

- 運の発生装置: **戦利品ドロップ**(レア度+ランダム効果の二重抽選)
- ゲームの本体: ドロップ品を**グリッド状のカバンに配置**し、形と隣接シナジーでビルドを組む
- 戦闘: **リアルタイム自動戦闘**(10〜30秒、倍速可)。プレイヤーは配置で戦い、戦闘は観る
- 大枠: **周回型 + 恒久成長のハイブリッド**(1周約20分、死んでも図鑑と解放は残る)

### 1.1 中毒性設計の対応表(本作の最重要要件)

| 報酬系の原則 | 本作での実装 |
|---|---|
| 変動報酬(いつ当たるか分からない) | ドロップのレア度抽選 / ショップリロール / 融合レシピの発見 |
| ニアミス(惜しい外れ) | 紫が光って橙じゃない演出 / HP残り数%での辛勝 / 予告された強敵への祈り |
| プッシュユアラック(賭けの上乗せ) | リロール課金 / 呪いの宝箱 / エリア間の「回復 or 追加ドロップ」二択 |
| 短ループ×長期成長 | 30秒戦闘 → 20分周回 → 恒久アンロック の三層 |
| 損失回避 | HP持ち越し制(ここで倒れたら周回が終わる緊張) |
| 自己帰属バイアス | 負け=ドロップ運のせいにできる / 勝ち=配置の腕と思える |
| 救済(不運による離脱防止) | 隠し天井(ピティ)システム |
| 収集・発見欲 | アイテム図鑑 / 隠し融合レシピのシルエット表示 |

## 2. コアループ

### 2.1 周回ループ(約20分)

```
周回開始(クラス選択・深淵Lv選択)
  └→ [ショップ/配置フェーズ] → [自動戦闘] → 勝利: ドロップ+G
        ↑                                      敗北: 周回終了
        └──────────── ×5戦 = 1エリア ──────────┘
  └→ エリアボス撃破 → 「HP40%回復 or 追加ドロップ2枠」の二択
  └→ 3エリア(森 → 遺跡 → 魔城)= 計15戦でクリア
```

### 2.2 恒久ループ

```
周回終了(クリア/敗北) → 魂片を獲得(到達戦闘数 × 深淵係数)
  → アンロック: 新アイテムをプールに追加 / 新カバン / 新クラス
  → クリア時: 深淵Lv+1 解放(敵強化 & 報酬強化)
  → 次の周回へ(「もう1周」の動機 = 新アイテムを引きたい + 深淵に挑みたい)
```

## 3. 戦闘システム

### 3.1 基本仕様

- **決定論的tickシミュレーション**(tick = 100ms)。seed付き乱数で同一入力→同一結果
  - 理由: テスト可能 / リプレイ可能 / ヘッドレスで1000戦回して自動バランス調整できる / 将来のデイリーシード機能に直結
- 双方のアイテムが**個別クールダウン(CD)**で自動発動。プレイヤー操作は倍速切替のみ
- 決着まで10〜30秒。60秒経過でサドンデス(双方に毎秒増加ダメージ)

### 3.2 リソース

| リソース | 仕様(初期値) |
|---|---|
| HP | 初期最大100。**戦闘後も持ち越し**(→ §8 回復設計) |
| スタミナ | 最大10、毎秒+1.0 回復。攻撃系はスタミナを消費。枯渇中は発動待機 |
| ブロック | 盾等で獲得するダメージ吸収スタック。上限30。戦闘終了でリセット |

### 3.3 効果の型(エンジンが解釈する語彙)

- **CD発動**: N秒ごとに効果(攻撃 / ブロック獲得 / 回復 / バフ)
- **トリガー発動**: `戦闘開始時` / `命中時` / `被弾時` / `ブロック時` / `敵撃破時` / `HP50%以下で` など
- **隣接効果**: 上下左右(または周囲8マス)の対象タイプに常時バフ。例: 砥石=隣接する武器に会心+10%
- **状態異常**: 毒(永続スタック、毎秒スタック分ダメージ)/ 火傷(3秒で消える高DoT)/ 鈍化(敵CD+20%)
- **会心**: 基礎5%、ダメージ2倍

### 3.4 初期数値のたたき台(実装後にシミュレータで調整)

| アイテム例 | サイズ | CD | 効果 | スタミナ |
|---|---|---|---|---|
| 短剣(白) | 1×1 | 1.2s | 4ダメージ | 0.5 |
| 剣(白) | 1×2 | 1.8s | 8ダメージ | 1.0 |
| 大剣(緑) | 2×3 | 3.5s | 24ダメージ | 3.0 |
| 木盾(白) | 2×2 | 2.2s | ブロック6獲得 | 0 |
| 薬草(白) | 1×1 | 10s | HP8回復(1戦闘1回) | 0 |
| 砥石(緑) | 1×1 | — | 隣接武器: 会心+10% | 0 |
| 毒瓶(青) | 1×1 | 2.5s | 毒2付与 | 0.5 |

※ 消耗品は作らない(全アイテム非消耗)。管理を単純化し、「配置=ビルド」に集中させる。

## 4. カバン(インベントリ)システム

- グリッドは **4×3 でスタート**、周回中にドロップする「カバン拡張」で最大 **6×5** まで成長
- アイテムは**ポリオミノ形状**(1×1〜2×3)。回転可。ドラッグ&ドロップで配置
- カバンに入らない分は**ストレージ**(戦闘に影響しない持ち物袋、上限8枠)へ。捨てる判断も戦略
- クラスごとに**初期カバンの形が違う**(欠けたマス配置)= クラスの個性その1

## 5. ドロップ&レア度システム(報酬系の心臓部)

### 5.1 レア度と抽選

| レア度 | 色 | 基礎確率 | 設計方針 |
|---|---|---|---|
| コモン | 白 | 50% | 基礎パーツ。融合の素材にもなる |
| アンコモン | 緑 | 30% | 隣接バフ・軽シナジー |
| レア | 青 | 14% | ビルドの軸候補(毒・会心・反射など) |
| エピック | 紫 | 5% | 軸の完成形。強トリガー持ち |
| レジェンダリー | 橙 | 1% | **数値でなくルールを歪める**(例: 「開幕時、隣接1アイテムを複製」「ブロックが上限を超えられる」) |

- エリアが進む & 深淵Lvが高いほど確率が上位側に傾斜
- **隠し天井(ピティ)**: 橙が30戦出ていなければ、以降1戦ごとに橙確率+1%。出たらリセット。**非公開仕様**
- ランダム効果(アフィックス): 青以上は追加効果を1〜2個抽選(例: 「+会心5%」「撃破時HP3回復」)。同名アイテムでも引きが違う = 二重抽選

### 5.2 ドロップ演出(工数を惜しまない最重要領域)

1. 敵撃破 → 宝箱落下 → **タメ(0.5s)** → レア度色の光柱 → アイテム表示
2. レア度別SE(橙は専用ファンファーレ)。紫以上は画面フラッシュ+振動風エフェクト
3. 開封は**タップで1個ずつ**(全部一括表示しない。「次は何だ」を毎回作る)

## 6. 融合レシピ(v1から実装)

- 特定の2アイテムを**隣接配置すると光ってヒント** → プレイヤーが承認すると融合(誤爆防止の承認制)
- レシピは**非公開・発見型**。図鑑に「?」シルエットで存在だけ示す → 実験欲を刺激
- v1目標: **20レシピ**

| レシピ例 | 結果 |
|---|---|
| 剣 + 砥石 | 名工の剣(会心20%内蔵) |
| 剣 + 聖水 | 聖剣(アンデッド特効) |
| 木盾 + 鉄鉱石 | 鉄盾(ブロック量2倍) |
| 薬草 + 薬草 | 上薬草(回復2倍・CD半減) |
| 毒瓶 + 短剣 | 毒牙の短剣(命中時 毒1) |

## 7. 敵設計 — 手作りアーキタイプ

方針: **敵は計15種(通常12+ボス3)を手作り**。各敵に明確な「攻め筋」と「対策軸」を持たせ、**次戦の敵を予告表示** → ショップでの対策購入という読み合いを作る。敵データはJSONで定義し、深淵LvでHP/攻撃に係数をかける。

| エリア | 敵 | 攻め筋 | 有効な対策軸 |
|---|---|---|---|
| 森 | ゴブリン | 高頻度の小攻撃 | ブロック・反射 |
| 森 | スライム | 低攻撃・高HP | DoT(毒)・持久 |
| 森 | 野盗 | 命中時にG窃盗(撃破で取り返す) | 速攻 |
| 森 | 狼 | 開幕3秒 攻撃2倍 | 開幕ブロック |
| 森 | **ボス: ゴブリンキング** | 手数+味方バフ | 反射・範囲 |
| 遺跡 | スケルトン | 1回だけHP50%で復活 | 高火力2連 |
| 遺跡 | 毒蜘蛛 | 毒スタック | 浄化・回復 |
| 遺跡 | 呪術師 | 開幕、プレイヤーの1アイテムを封印 | 冗長性のあるビルド |
| 遺跡 | ガーゴイル | 高ブロック | 貫通・DoT |
| 遺跡 | **ボス: 石像巨人** | 超装甲・遅い大打撃 | 貫通・鈍化 |
| 魔城 | 暗黒騎士 | 被弾時 反射ダメージ | 大打撃少数・DoT |
| 魔城 | ヴァンパイア | 与ダメ吸血 | バースト・火傷 |
| 魔城 | リッチ | プレイヤーのスタミナ回復-50% | 低スタミナ構成 |
| 魔城 | 炎の悪魔 | 火傷DoT | 回復・耐火 |
| 魔城 | **ボス: 魔王** | 2フェーズ(装甲→狂乱) | 総合力 |

## 8. 周回構造・経済・回復設計

### 8.1 進行

- 1周 = **3エリア × 5戦**(5戦目がボス)。想定20分
- 各戦闘前に**ショップ/配置フェーズ**(時間無制限)

### 8.2 ショップと経済

- 勝利報酬: ドロップ2枠 + ゴールド(10〜20G、エリアで増加)
- ショップ: 6枠陳列。**リロール 5G → 7G → 10G…**(ショップごとにリセット)← 小さなギャンブル
- 売却: 購入額の半値。カバン圧迫時の損切り判断
- **呪いの宝箱**(ショップに低確率出現): 高レア確定だがデメリット付き(例: 「強力だが隣接マスを呪う」)
- **ギャンブル商人**(低確率出現): 30G払うとレア度完全ランダムの袋

### 8.3 回復設計(HP持ち越し制の要)

通常勝利では回復しない。回復手段を**有限の選択**にして緊張を維持する:

1. **エリアボス撃破後の二択: 「最大HP40%回復」or「追加ドロップ2枠」** ← 欲張るか、安全を取るか
2. 回復系アイテム(薬草など)をカバンに積む = 火力枠との交換
3. 吸血・撃破時回復ビルド
4. ショップの回復サービス(20GでHP20)

## 9. 恒久成長(メタプログレッション)

- メタ通貨: **魂片** = 到達戦闘数 × 深淵係数(敗北でも貰える)
- **多様性を解放する**(恒久的なステータス強化は入れない):
  - 新アイテムのプール追加(図鑑から**自分で選んで**解放 = 自己決定感)
  - 新カバン形状 / 新クラス
- クラス(v1で3種): 初期装備と初期カバン形状のみ差別化
  - 戦士(剣+木盾 / 正方形寄りカバン)
  - 狩人(短剣×2 / 横長カバン)
  - 薬師(薬草+毒瓶 / 変形カバン)
- **深淵レベル**(クリアごとに+1解放、最大10): 敵HP+15%/Lv・攻撃+10%/Lv、代わりにレア度傾斜と魂片係数+20%/Lv
  - 「強くなった分だけ坂も急になる」= 緊張の永続化

## 10. 画面構成

1. **ホーム**: 周回開始 / 図鑑 / 解放(魂片) / 設定
2. **出撃準備**: クラス選択・深淵Lv選択
3. **メイン画面**(1画面統合): 上=進行度+次敵予告 / 中=戦闘ビュー(HP・発動ログ) / 下=カバングリッド+ストレージ
4. **ショップ**: メイン画面下部に陳列が出る形(配置しながら買える)
5. **リザルト**: 到達記録 / 獲得魂片 / 図鑑更新ハイライト

レスポンシブ対応(PC優先、スマホは縦1カラム)。

## 11. 技術スタック

| 領域 | 選定 | 理由 |
|---|---|---|
| ビルド | Vite + React 18 + TypeScript | 静的SPAで十分。Vercelに最適 |
| 状態管理 | Zustand | 軽量、ゲーム状態と相性良 |
| D&D | dnd-kit | グリッド配置の実績多数 |
| 演出 | Framer Motion + CSS | 光柱・タメ演出 |
| 音 | Howler.js | レア度別SE |
| 戦闘エンジン | 純TSモジュール(UI非依存) | ヘッドレステスト・自動バランス調整 |
| 乱数 | mulberry32(seed付き) | 決定論・リプレイ・将来のデイリーシード |
| テスト | Vitest | エンジンの回帰テスト |
| セーブ | localStorage + JSONエクスポート/インポート | v1はサーバ不要 |
| デプロイ | **Vercel**(全部) | — |
| **Render** | v2以降: クラウドセーブ / デイリーシード配信 | v1では使わない |

### リポジトリ構成案

```
/src
  /engine      # 戦闘シミュレータ(UI非依存・テスト対象)
  /data        # items.json / enemies.json / recipes.json  ← データ駆動
  /ui          # React コンポーネント
  /store       # Zustand
/docs
  DESIGN.md    # 本書
/tests
```

**データ駆動を徹底**: アイテム・敵・レシピは全てJSON定義。バランス調整=JSONの数字いじりで完結させる。
