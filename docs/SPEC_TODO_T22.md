# SPEC_TODO T22 — 追加ショップイベントの未確定事項

## TASKS T22: 呪いの宝箱の紫以上抽選

- **箇所**: T22は「紫以上」を要求するが、エピックとレジェンダリーの内訳を指定していない。
- **問題**: 等確率、固定比率、エリア・深淵依存のどれを採用するかで報酬期待値が変わる。
- **暫定判断**: 既存のショップ用レア度分布をエピック以上へ条件付けし、相対比を維持する。dropLuckと天井は従来どおりショップへ適用しない。
- **理由**: 新しいバランス値を追加せず、進行度と深淵による既存のショップ傾斜を再利用できるため。
- **今後の確認**: 専用比率を正式採用する場合は`config.json`へエピック・レジェンダリー比率を追加し、固定シードテストを更新する。

## TASKS T22: イベント判定とリロール

- **箇所**: 「出現判定がshopストリームで決定論的」とあるが、リロールごとに再判定するか、ショップ入場時に1回だけ判定するかが未指定。
- **問題**: リロール再判定ではゴールドを払ってイベントを狙える一方、ショップ単位固定では通常陳列のリロールとイベントが独立する。
- **暫定判断**: `fork(runSeed, "shop:{battleIndex}:events")`でショップ単位に固定し、通常陳列の`shop:{battleIndex}:{rerollCount}`とは分離する。購入済み状態はリロールとリロードを跨いで保持する。
- **理由**: リロールで10%判定を反復できる抜け道を防ぎ、同じショップを閉じて開き直しても結果を再現できるため。
- **今後の確認**: リロールによるイベント探索を正式なゲーム性にする場合は、出現率と経済期待値を再設計する。

## TASKS T22: 特別イベントの同時出現

- **箇所**: 呪いの宝箱とギャンブル商人が同じショップに同時出現できるか未指定。
- **問題**: 排他にすると二つ目の判定方法と優先順位が必要になる。
- **暫定判断**: 二つを独立した10%判定とし、同時出現を許可する。それぞれ1回だけ購入可能。
- **理由**: configの各出現率をそのまま解釈でき、優先順位という新規仕様を導入しないため。
- **今後の確認**: 特別枠を1枠へ制限する場合は、競合時の選択規則を正式定義する。

## TASKS T22: 呪いアフィックスのデータ境界

- **箇所**: 最大HP-10は呪いの宝箱専用だが、通常アフィックスプールへ入れるかは未指定。
- **問題**: 通常プールへ追加すると通常ドロップにも呪いが抽選され、T22の専用デメリットではなくなる。
- **暫定判断**: 数値は`config.json`を参照し、獲得アイテムの`resolvedTriggers`へbattleStartのmaxHp負値として保存する。通常アフィックスプールには追加しない。
- **理由**: 既存の戦闘・セーブ境界を使いながら、呪いを特別イベント専用に限定できるため。
- **今後の確認**: 将来、呪いを一般カテゴリとして展開する場合は専用affix IDと通常抽選からの除外フラグをスキーマへ追加する。
