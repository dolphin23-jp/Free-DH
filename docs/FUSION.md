# FUSION — 融合フロー

T19で実装したカバン内融合の境界を定義する。レシピの正本は `src/data/recipes.json`、アイテム定義は `src/data/items.json` とする。

## 成立条件

- 素材は両方ともカバン内の物理アイテムであること。
- 2つの占有マスが上下左右のいずれかで接していること。斜め接触は対象外。
- `recipes.json` の `a` / `b` と一致すること。素材順は不問。
- 同一素材レシピでは、別々のインスタンスが2個必要。

`findFusionCandidates` は候補を列挙するだけでインベントリを変更しない。カバンUIは候補に含まれるアイテムを発光させ、候補一覧を表示する。

## 確認と実行

候補を選択すると、素材2個と完成品を示す確認ダイアログを開く。キャンセル、配置変更、ダイアログ外の通常操作では素材を消費しない。

「融合を確定」を押した場合だけ `fuseInventory` を実行する。実行直前に以下を再検証する。

1. 両素材がまだカバン内に存在する。
2. 2つが別インスタンスである。
3. レシピと完成品が候補作成時から変わっていない。
4. 両素材が現在も上下左右に隣接している。
5. 完成品を置ける空きマスがある。

この再検証により、古い確認状態からの誤融合を防ぐ。

## 完成品の配置

素材2個を一時的に除いた後、以下の順で未回転の完成品を配置する。

1. 素材の左上座標をグリッド順に試す。
2. カバン全体を行優先・左から右へ走査する。
3. どこにも入らない場合は融合せず、素材を保持したままエラーにする。

ストレージへの自動退避や自動回転は行わない。

## 完成品と図鑑

完成品は新しい物理インスタンスとして作成し、素材のアフィックスと `runDamageBonus` は継承しない。成功時に完成品アイテムと実行したレシピを図鑑へ登録する。

融合は乱数を使用せず、`Math.random` も使用しない。

## テスト境界

- R01〜R20を素材順の両方向で照合する。
- R01〜R20を隣接配置から実際に融合できることを確認する。
- 候補検出だけではインベントリが変化しないことを確認する。
- 非隣接素材を候補にしない。
- 候補作成後に素材を移動した場合、実行を拒否する。
