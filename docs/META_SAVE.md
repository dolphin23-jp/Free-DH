# META_SAVE — メタ進行とセーブ

T20で追加した恒久進行、深淵補正、自動保存の境界を定義する。バランス数値は `src/data/config.json`、戦闘規則は `docs/COMBAT_SPEC.md` を正とする。

## メタ進行

恒久状態は周回状態と分離し、以下を保存する。

- 所持魂片
- 恒久アンロック済みアイテムID
- 選択可能な最大深淵Lv
- 魂片受取済み周回キー

初期状態では `fusionOnly: false` かつ `unlockCost: 0` のアイテムだけが通常のドロップ・ショップ候補となる。融合専用品はアンロックショップや通常抽選へ入らない。

アンロック時はアイテム自身の `unlockCost` を消費する。同じアイテムを再購入して魂片を重複消費することはない。

## 魂片と深淵解放

周回リザルトの獲得魂片は次式で算出する。

```text
ceil(到達戦闘数 × (1 + 0.2 × 深淵Lv)) + クリア時10
```

例:

- 深淵0を15戦クリア: `ceil(15 × 1.0) + 10 = 25`
- 深淵2で10戦目到達: `ceil(10 × 1.4) = 14`

リザルト報酬は `runSeed + outcome + wins + reachedBattleCount` から作る受取キーで一度だけ加算する。画面再描画、リロード、同一セーブの再読込で二重加算しない。

深淵Lvをクリアすると、設定上限まで次のLvを1段階解放する。未解放Lvは周回開始時に選択できない。

## 深淵補正

深淵Lv `L` の敵には、元データから毎戦以下を適用する。

```text
HP倍率 = 1 + 0.15 × L
攻撃倍率 = 1 + 0.10 × L
```

深淵Lv3以上では、各エリアの通常戦4枠からシード固定で1戦を強化個体にする。ボスは強化個体へ置換しない。

強化個体にはさらにHP `+30%` を乗算し、通常のドロップ数へ1枠追加する。強化個体の選出は `fork(runSeed, "elite:{area}")` から決定し、同じ周回シードでは一致する。

敵定義は補正前のJSON由来データへ毎回戻してから倍率を適用する。深淵Lvの変更や同じ敵との再戦で補正が累積しない。

## セーブversion 2

統合セーブは以下を含む。

- `run`: 周回フェーズ、HP、所持G、カバン、ストレージ、ボス報酬、リザルト
- `codex`: アイテム、敵、融合レシピの発見状況
- `meta`: 魂片、アンロック、深淵解放、報酬受取履歴
- `dropProgress`: 橙天井、未処理宝箱、予約済みボス追加報酬
- `shop`: 現在の陳列、購入済み枠、回復利用状況

読込時は全セクションを検証してからライブストアを更新する。途中で検証に失敗した場合は、既存のゲーム状態を部分的に変更しない。

version 1の統合セーブはversion 2へ移行する。旧セーブに存在しないメタ進行、ドロップ進行、ショップ状態は初期値で補完する。旧run snapshot version 1は、深淵0・ボス報酬なしとしてversion 2へ移行し、リザルト中なら魂片を再計算する。

## 保存方法

ブラウザでは `free-dh:game-save:v2` へ完全なゲーム状態を自動保存する。周回、図鑑、メタ、ドロップ、ショップのいずれかが変化すると保存を更新する。

ホーム画面では現在状態をJSONとして出力でき、貼り付けたJSONを検証・移行した上で読み込める。読込成功後は新しいversion 2形式で自動保存し直す。
